# KamboGuide — Mobile (PWA + native apps)

KamboGuide ships two ways from one codebase:

1. **Installable PWA** (web) — offline-capable, add-to-home-screen, web push.
2. **Native iOS + Android apps** via **Capacitor** (same web build wrapped in a native shell for the App Store / Play Store).

---

## 1. PWA (already live on the web build)

- **Service worker**: `src/sw.ts` (built to `dist/sw.js` by `vite-plugin-pwa`, `injectManifest`). Precaches the app
  shell (offline), runtime-caches images/fonts (cache-first) and Supabase GETs (network-first), and handles **web push**
  + notification clicks. Auto-registers on load.
- **Manifest**: `public/manifest.webmanifest` (standalone, PNG + maskable icons, categories). Icons generated by
  `@vite-pwa/assets-generator` from `public/icon.svg` — regenerate with `node node_modules/.bin/pwa-assets-generator`.
- **Install prompt**: `src/components/InstallPrompt.tsx` (Android/desktop `beforeinstallprompt` + iOS "Add to Home Screen" hint).
- **Push**: `src/lib/push.ts` (subscribe) → `src/lib/notify.ts` → `api/push-send.js` (Vercel, `web-push`).

### Web push — activate in production
Keys are generated locally in `.env`. In **Vercel → Project → Settings → Environment Variables** set:
- `VITE_VAPID_PUBLIC_KEY` (public — same as `.env`)
- `VAPID_PRIVATE_KEY` (secret)
- `VAPID_SUBJECT` (e.g. `mailto:239hive@gmail.com`)
Then redeploy. Users enable push in **Account → Security**.

## 2. Native apps (Capacitor)

App id **`com.kamboguide.app`**, name **KamboGuide**. Config: `capacitor.config.ts`. Native init (status bar, splash,
Android back button, deep links, haptics): `src/lib/native.ts` (called from `main.tsx`, no-ops on web).

- **Android** project is generated (`android/`).
- **iOS** project is generated (`ios/`) and **verified building + running in the iOS Simulator** (iPhone 17 Pro).
  Deployment target is **iOS 15.0** (Capacitor 8 minimum) — set in both `ios/App/Podfile` and the Xcode project.

### Dev workflow (every time web code changes)
```bash
npm run build                                   # build the web app → dist/
node node_modules/.bin/cap sync                 # copy dist + plugins into native projects
node node_modules/.bin/cap open android         # → Android Studio (run/emulator)
node node_modules/.bin/cap open ios             # → Xcode (after iOS is added)
```
(`npx cap …` also works on machines without the `:` in the repo path.)

### Run on the iOS Simulator (verified working)
```bash
brew install cocoapods                          # one-time (already installed here)
node node_modules/.bin/cap sync ios             # after any web change
# Then open in Xcode and press ▶, or run headless:
xcrun simctl boot "iPhone 17 Pro"
xcodebuild -workspace ios/App/App.xcworkspace -scheme App -sdk iphonesimulator \
  -configuration Debug -derivedDataPath /tmp/kg-ios-build \
  -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build
xcrun simctl install booted /tmp/kg-ios-build/Build/Products/Debug-iphonesimulator/App.app
xcrun simctl launch booted com.kamboguide.app
```
`ios/App/Podfile` **and** the Xcode project are pinned to **iOS 15.0** (Capacitor 8 minimum). A fresh `cap add ios`
defaults to 14.0 and will fail to compile until both are bumped to 15.0.

### App icons + splash for native
```bash
npm i -D @capacitor/assets
npx @capacitor/assets generate --iconBackgroundColor '#faf8f4' --splashBackgroundColor '#2f5e46'
```
(Provide `assets/icon.png` 1024×1024 + `assets/splash.png` 2732×2732 for best results.)

### Store submission
- **iOS**: Apple Developer account ($99/yr). In Xcode: set Team + signing, bump version/build, Product → Archive →
  Distribute → App Store Connect. Add screenshots + listing in App Store Connect.
- **Android**: Google Play Console ($25 one-time). In Android Studio: create a signing keystore, Build → Generate Signed
  Bundle (`.aab`), upload to Play Console with listing + screenshots.
- Privacy policy is live at `/Privacy`; link it in both store listings.

## Native OAuth deep-link — WIRED ✅ (remaining: dashboard config)
Native Google/GitHub sign-in: `src/pages/Auth.tsx` opens the provider in the system browser (custom scheme
`com.kamboguide.app://oauth`); `src/lib/native.ts` catches the deep link back and completes login via
`exchangeCodeForSession` (PKCE — set in `src/lib/supabase.ts`); the Android URL scheme is registered in
`android/app/src/main/AndroidManifest.xml`. Email/password already worked. To finish:
- **Supabase → Auth → URL Configuration → Redirect URLs**: add `com.kamboguide.app://oauth` **and**
  `com.kamboguide.app://reset` (password-reset deep-link, also wired — Auth.tsx + native.ts).
- **iOS** (after `cap add ios`): add the scheme to `ios/App/App/Info.plist` (`CFBundleURLSchemes` = `com.kamboguide.app`).
- Ensure Google/GitHub providers are enabled in Supabase Auth.

## Native push (FCM/APNs) — client WIRED ✅ (remaining: provider infra)
`src/lib/nativePush.ts` (called from `native.ts`) requests permission, registers the device, and stores the token in
`push_subscriptions` (tagged `keys.platform`); notification taps deep-link into the app. Local notifications
(`@capacitor/local-notifications`) work now. To actually DELIVER remote push:
- **Android**: create a Firebase project, add `google-services.json` to `android/app/`, enable FCM.
- **iOS**: create an APNs auth key + enable the Push Notifications capability in Xcode.
- **Send path is built**: `api/native-push-send.js` (FCM HTTP v1) sends to native tokens; `src/lib/notify.ts` routes
  native subs there and web subs to `api/push-send.js`. **Activate by setting `FCM_SERVICE_ACCOUNT`** (the Firebase
  service-account key JSON) in Vercel env. (iOS-via-FCM also needs the APNs auth key uploaded in Firebase.)
